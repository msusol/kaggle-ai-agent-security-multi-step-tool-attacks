## Remaining issue: Phase B timing overrun

Phase B's internal `ok(margin=5)` lets probes start if `elapsed + 5 < time_limit`. The last Gen 0
beam expansion finished 14s after the deadline (Gemma call in flight when the timer expired).
This overrun reduces D's actual time to `39.7 - 14 = 25.7s` effective.

At the 9000s Kaggle budget, B's time_limit = min(3600, ~7200 - 600 - 1800) = 4800s and Phase D
gets 1800s. A 14s overrun is negligible. Only relevant at 300s DGX budget.

**Options for tighter DGX timing:**
1. Increase Phase B's `ok()` margin from 5s to 15s (prevents late probe starts)
2. Adjust budget fractions: B: 0.35 → D: 0.25 (gives D 67.5s vs 54s at 300s budget)
